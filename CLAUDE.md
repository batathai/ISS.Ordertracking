# CLAUDE.md — บริบทโปรเจกต์ ISS Order Tracker (BATA)

> ไฟล์นี้สรุปทุกอย่างที่ต้องรู้ก่อนแก้โค้ดในโปรเจกต์นี้ อ่านไฟล์นี้ก่อนเริ่มงานทุกครั้ง

## 1. ภาพรวมโปรเจกต์
เว็บแอปติดตามสถานะออเดอร์/พัสดุ สำหรับสาขาของ BATA (แบรนด์รองเท้า) ชื่อแอป **"BATA ISS · Track"**
- สาขาเปิดเว็บดูสถานะออเดอร์ + เลขพัสดุของตัวเองได้ (ค้นหา, ดูรายละเอียด, คัดลอกเลขพัสดุ)
- แอดมิน (เข้าผ่าน `?admin=1` ใน URL) นำเข้าไฟล์ Excel/CSV ออเดอร์ใหม่เข้าระบบ และดูประวัติการนำเข้า (Log)
- ข้อมูลหลักเก็บใน **Google Sheets** อ่านผ่าน CSV export URL, เขียน/import ผ่าน **Google Apps Script Web App**

## 2. สถานะปัจจุบัน
**ใช้งานจริงแล้ว (production)** — สาขาใช้งานอยู่ทุกวัน ผ่านการดีบั๊กหนักในสัปดาห์ที่ผ่านมาจนเสถียรแล้ว (ดู Known Issues ที่แก้ไปแล้วด้านล่าง)

## 3. โครงสร้างไฟล์
```
/ (root ของ repo — deploy ผ่าน GitHub Pages)
├── CNAME          ← custom domain สำหรับ GitHub Pages
└── index.html     ← ไฟล์เดียวทั้งแอป (HTML+CSS+JS inline ทั้งหมด ไม่มีไฟล์แยก)
```
**ไม่มี** Apps Script code เก็บอยู่ใน repo นี้ (อยู่แยกใน Google Apps Script Editor ของ Google Sheet เอง ไม่มีการ backup โค้ดไว้ที่ไหนอย่างเป็นทางการ — ควรพิจารณาก็อปมาเก็บไว้ใน repo นี้ด้วยในอนาคต)

## 4. Tech Stack
- Vanilla JS + CSS ล้วน (ไม่มี framework, ไม่มี build step)
- Deploy ผ่าน **GitHub Pages** (repo: `batathai/ISS.Ordertracking`) — แก้ไฟล์แล้ว push ขึ้น `main` ก็ deploy อัตโนมัติ
- Library ภายนอกตัวเดียว: [SheetJS (xlsx.full.min.js)](https://cdnjs.cloudflare.com) สำหรับอ่านไฟล์ Excel/CSV ตอน import
- Backend: Google Sheets (เก็บข้อมูล) + Google Apps Script Web App (endpoint สำหรับ import/log)
- ข้อมูลสถานะ override + ประวัติการเปลี่ยนสถานะ เก็บใน `localStorage` ของเบราว์เซอร์ผู้ใช้ (คีย์ `bata_ov`, `bata_hi`)

## 5. Key Decisions / เหตุผลการออกแบบ
- **อ่านข้อมูลจาก Google Sheets ผ่าน CSV export URL** (ไม่ใช้ Sheets API) เพราะง่าย ไม่ต้องมี auth ฝั่ง client
- **เขียน/import ผ่าน Apps Script Web App** แยกจากการอ่าน เพราะ CSV export เป็น read-only
- **ซิงค์แบบ manual เท่านั้น** (เปิดแอป + กดปุ่ม ↻ รีเฟรช) — ไม่มี auto-refresh พื้นหลัง เพื่อประหยัด Google Sheets quota และแบตมือถือ (เป็นการตัดสินใจร่วมกับผู้ใช้)
- **แสดงผลเฉพาะ 2 เดือนล่าสุด** (คำนวณจากคอลัมน์วันที่) เพื่อไม่ให้ข้อมูลเก่าท่วมหน้าจอ
- **เมนูแอดมิน (import/log) ซ่อนอยู่หลัง `?admin=1`** สาขาทั่วไปมองไม่เห็นเมนูนี้เลย
- **Import ซ้ำไฟล์เดิมได้อย่างปลอดภัย** — ระบบจับคู่ด้วย Order2+SKU เดิม ถ้ามีอยู่แล้วจะอัปเดต ไม่สร้างซ้ำ ออกแบบมาเพื่อให้กด import ซ้ำเวลาต้องการรีเฟรชสถานะได้เลยโดยไม่ต้องกลัวข้อมูลซ้อน

## 6. จุดสำคัญมากในโค้ด (ห้ามพลาดถ้าจะแก้)

### 6.1 คอลัมน์ Google Sheet (ตัวแปร `COL` ใน index.html)
```js
const COL = { order1:0, order2:1, sku:2, date:3, shop:4, remark:5, status:6, rm:7, tracking:8, doc:10, month:11 };
```
⚠️ **`status` อยู่ที่ index 6** ไม่ใช่ index 9 — เคยเข้าใจผิดมาก่อนแล้วทำให้ข้อมูลสถานะหายไปหมด ถ้า column ใน Sheet ถูกสลับ/เพิ่ม ต้องแก้ตรงนี้ก่อนอย่างอื่นทั้งหมด

### 6.2 STATUS_TEXT_MAP — การแปลสถานะข้อความไทยจาก Sheet
ระบบต้อง map ข้อความไทยที่ import มาให้ตรงกับ 4 สถานะที่แอปแสดง (`wait`/`prepare`/`transit`/`cancel`) ผ่าน `STATUS_TEXT_MAP` ถ้า Sheet มีข้อความสถานะแบบใหม่ที่ไม่อยู่ใน map นี้ จะ fallback เป็น "รอดำเนินการ" เงียบๆ โดยไม่ error — ต้องเช็คทุกครั้งที่เพิ่มสถานะใหม่

### 6.3 Google Sheets Data Validation จำกัดค่าที่เขียนได้
คอลัมน์ Status ในชีตมี Data Validation จำกัดไว้แค่ 6 ค่าตายตัว ฝั่ง Apps Script (นอก repo นี้) ต้อง:
1. เช็คค่าก่อนเขียนว่าตรงกับ 6 ค่าที่อนุญาตไหม ถ้าไม่ตรงให้ข้าม ไม่เขียนทับ
2. ห่อการเขียนแต่ละแถวด้วย try/catch แยกกัน ไม่ให้ 1 แถวพังแล้วทำทั้งไฟล์หยุดกลางคัน

### 6.4 CSV Parser แบบ custom (`parseCSVRows_`)
เขียนขึ้นมาเองแบบอ่านทีละตัวอักษร เพื่อรองรับเซลล์ที่มีข้อความขึ้นบรรทัดใหม่ในตัว (เช่น remark ที่กด Enter) **ห้ามเปลี่ยนกลับไปใช้ `text.split('\n')` ธรรมดา** เพราะเคยทำให้ข้อมูลทั้งไฟล์เพี้ยนตั้งแต่จุดที่มีบรรทัดใหม่ในเซลล์เป็นต้นไป

### 6.5 Order2 ต้อง pad เป็น 13 หลัก
ฟังก์ชัน `padOrder2()` เติม 0 นำหน้าให้ครบ 13 หลักเสมอ (เช่น `100624009` → `0000100624009`) เพื่อให้สาขา search เจอง่าย — ถ้า Google Sheets แปลงเป็น Number แล้วตัด 0 นำหน้า ต้องเช็คจุดนี้

## 7. Known Issues ที่แก้ไปแล้ว (ประวัติบั๊กสำคัญ อย่าทำซ้ำ)
- **Mapping สถานะภาษาไทยไม่รู้จัก** → เพิ่ม `STATUS_TEXT_MAP` แล้ว
- **คีย์จับคู่ออเดอร์ซ้ำพัง** เพราะ Google Sheets แปลงเลขเป็น Number ตัด 0 นำหน้า → แก้ด้วย `padOrder2()`
- **Import เขียนผิดแท็บ** ในชีต (แท็บซ้ายสุด ≠ แท็บที่เว็บอ่านจริง) → ต้องเช็ค gid ใน `SHEET_CSV` ให้ตรงกับแท็บจริงเสมอ
- **คอลัมน์ Status จริงอยู่ index 6** ไม่ใช่ 9 ตามที่เคยเข้าใจผิด
- **`parseCSV` เดิมพังกับเซลล์ที่มีข้อความขึ้นบรรทัดใหม่** → เขียน `parseCSVRows_` ใหม่แบบอ่านทีละตัวอักษร
- **Apps Script deployment แคชค้าง** — บางครั้งต้องสร้าง deployment ใหม่ทั้งหมด (ไม่ใช่แค่ New version) ถ้าโค้ดที่แก้ไม่มีผลจริงบนเว็บ
- **Data Validation ของคอลัมน์ Status จำกัดค่า** ทำให้ import พังกลางคันถ้าเจอค่าที่ไม่ตรง dropdown → แก้ด้วยการเช็คค่าก่อนเขียน + try/catch แยกแถว

## 8. วิธี Deploy
ไม่มี build step ใดๆ — แก้ `index.html` แล้ว push ขึ้น branch `main` ของ repo `batathai/ISS.Ordertracking` GitHub Pages จะ deploy อัตโนมัติภายใน 1-2 นาที (custom domain กำหนดผ่านไฟล์ `CNAME`)

## 9. Apps Script (อยู่นอก repo นี้)
- URL ปัจจุบันอยู่ในตัวแปร `APPS_SCRIPT_URL` ใน index.html
- ทำหน้าที่: รับ POST `{action:'import', filename, rows}` เพื่อเขียน/อัปเดตแถวใน Sheet, และ GET `?action=log` เพื่อดูประวัติการนำเข้า
- **ควรพิจารณา backup โค้ด Apps Script มาเก็บเป็นไฟล์ในโฟลเดอร์นี้ด้วย** เพื่อป้องกันการทำ CLAUDE.md/โค้ดไม่ตรงกับของจริงในอนาคต
